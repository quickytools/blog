Firing Up Push to Deploy

Time escapes through a wormhole

when you spin on a gerbil wheel. There is nothing more wasteful than having to perform the same tasks again and again. Isn’t software supposed to handle the medium that becomes tedium? Combine Gitlab and Firebase for the ultimate build pipeline when you develop the next greatest app and have the world like 😮

👷 the piping

  1. Create new projects
    1. Use Vue CLI and create a Vue project
    2. Create a Firebase project
    3. Create a new project on Gitlab
  2. Push the initial commit to Gitlab (set git remote origin as necessary)
  3. Initialize Firebase in the Vue project
    1. The screenshot shows Firebase Functions is included during creation which requires additional effort in the CI script later
    2. If your Firebase project doesn’t show in the select default step choose don't setup a default project and manually set it later
    3. Remember the public directory (dist in screenshot) for the CI script later
  4. Prep CI+CD
    1. Add deploy command to package.json
    2. Create .gitlab-ci.yml similar to this
      1. cache loads node_modules and functions/node_modules before each job if caches exist
      2. build-production installs dependencies and builds the Vue app for release
      3. test-production runs unit tests
      4. deploy-production deploys to Firebase hosting
      5. artifacts are outputs of a job and dependencies are previous jobs whose artifacts are used
      6. artifacts must set paths to the public directory (dist in screenshot)
  5. Commit and push the changes
    1. GitLab CI should pass the build and test jobs but fail the deploy job like follows
    2. The error in the screenshot is Too many arguments because a variable is not yet set
  6. Retrieve and set a Firebase CI token
    1. If the Firebase project is not set do so with firebase use <project-id> where <project-id> can be found on the Firebase console
    2. Follow instructions after firebase login:ci for a CI token
    3. Set the token under the project’s Settings > CI/CD > Environment Variables FIREBASE_DEPLOY_TOKEN
  7. Rerun the deploy job only like pressing Retry on #162819683
    1. The pipeline should run to completion successfully and deploy to your Firebase hosting instance

Now go forth and set the 🌎 on 🔥🔥🔥
