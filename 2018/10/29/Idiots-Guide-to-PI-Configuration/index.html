Idiots Guide to Pi Configuration

Beware of information hidden

in plain sight. A no-parking sign in a poorly lit area at night is fair warning for a tow truck to drag your unwilling vehicle to the yards. Let’s not even get to written phone camera rules when you come to bail it out the next day…

  1. Flash a micro SD card using Etcher with a raspian image
    1. If necessary unplug and replug the SD adapter so the computer recognizes it
  2. In the /boot directory create files
    1. wpa_supplicant.conf and specify your network
      1. If you edit it using a text editor by renaming it with a .txt extension be sure to revert the name
      2. When reverting be sure to show file extensions or OSX will show wpa_supplicant.conf when it might really be wpa_supplicant.conf.txt
    2. Empty file named ssh
  3. Eject the SD card safely and plug it into a Pi with wireless capabilities
  4. Power up the Pi and give it time to boot
  5. SSH in with `pi@raspberrypi.localpwraspberry`
    1. Change the account password with passwd pi
    2. Setup passwordless SSH by copying an SSH key with ssh-copy-id …
  6. 🔮 🍓

Troubleshooting

Setup fails if wpa_supplicant.conf exists in the /boot directory when the SD card is plugged back into a computer. Verify the network credentials and make sure the file is named wpa_supplicant.conf with extensions shown.
